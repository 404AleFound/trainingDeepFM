## Common Metric
### AUC
- **含义**：ROC 曲线下侧面积（Area Under the ROC Curve）。在CTR场景中，AUC代表真实的点击样本其预测概率高于未点击样本预测概率的比例。
- **优点**：**CTR领域最重要的评估指标**。对样本类别不平衡（CTR任务中正样本/点击极少）极其鲁棒；它关注的是模型对样本的**相对排序能力**，而非绝对概率值的偏置，这非常契合真实的广告竞价排序或推荐流排序逻辑。
- **缺点**：不能反映预测概率绝对值的精准度。例如将所有预测结果都乘0.5，AUC会保持原样不变，但在需要利用真实估算CTR进行扣费、出价补偿时，这种绝对值偏差会引发严重业务问题。

### LogLoss
- **含义**：对数损失（Logarithmic Loss / Cross Entropy Loss），衡量预测概率分布与真实标签分布之间的差异。
- **优点**：不仅要求模型排序正确，更要求**预估的绝对概率值准确（Calibration）**。在RTB（实时竞价）广告场景中，最终的出价通常由 `预估CTR * 目标出价` 算出，因此精准的 pCTR 值至关重要，此时 LogLoss 是不可或缺的监控指标。
- **缺点**：对异常值（Outliers）极为敏感；而且 LogLoss 的基线会受到数据集的正负样本比例极大的影响。如果背景大盘的CTR天然极低，LogLoss的绝对值也会比较小，这使得不同分布或不同场景的数据集之间难以直接横向对比。

### ACC
- **含义**：准确率（Accuracy），即预测正确的样本数占总样本数的比例（通常设定阈值为0.5）。
- **优点**：定义极其直观，所有人都容易理解。
- **缺点**：**极其不适合CTR场景**。由于点击率往往极其不平衡（例如只有 1% 用户点击），模型只要采取“偷懒”策略——将所有样本无脑预测为“不点击”（0），其准确率也能高达99%。此时模型其实彻底失效，但ACC指标却看似完美，具有巨大的误导性。

### Pre
- **含义**：精准率（Precision），在所有被模型预测为正样本（点击）的请求中，真实发生点击的比例。
- **优点**：关注预测的“质量”。在CTR场景下，它反映了“推出去的高优广告中，用户实际点击的情况”，预示了转化质量和每次展现收益潜力。
- **缺点**：容易通过大幅度拔高判定阈值（只对极其确定的少数点击进行推荐）来刷高该指标。但这会导致展现量急剧萎缩，错失大量长尾请求中的真实潜在点击。

### Rec
- **含义**：召回率（Recall），在所有的真实正样本（实际发生点击的请求）中，被判定超过阈值并由模型成功预测出来的比例。
- **优点**：关注模型的“覆盖度”。它代表模型有没有漏掉真正想点击的用户。
- **缺点**：同样可以通过降低阈值（不管三七二十一，把大量请求都圈定为高潜点击）来提高。这会带来海量的“无效判定”，浪费计算与展示资源，严重拉低广告整体CTR与点击转化率。

### F1
- **含义**：F1-Score，即 Precision 和 Recall 的调和平均数，致力于同时兼顾准确率与召回率。
- **优点**：在正负样本极为不均衡时，相比孤立的 Pre、Rec 具有更好的平衡与客观评价作用，有效防止了极端偏袒某一侧的弊端。
- **缺点**：无论是 ACC、Pre、Rec 还是 F1，仍然都强依赖一个指定的“硬切分阈值”（如0.5）将模型输出硬切为 0 或 1。由于广告和推荐系统本质是排序分发系统，强行设定阈值容易截断连续分布的信息。因此它在工业界往往用作辅助参考，不如 AUC 直观和常用。

### LR
- **含义**：学习率（Learning Rate），通常代表模型在训练时参数更新的步长大小（注：有时也泛化地表示 Logistic Regression 逻辑回归，但在 Metric 语境中通常指学习率监测）。
- **优点**：作为训练过程的超参数/监控变量，实时将训练Loss与LR曲线结合观察，能有效帮算法人员判断目前网络所处状态。CTR大规模稀疏特征模型多采用自适应优化器结合特定 Scheduler，监控 LR 便于排查是不是因为LR衰减导致训练提早停滞。
- **缺点**：仅作为训练“过程观测”指标，与模型在具体业务上的预测性能和泛化能力（如AUC、LogLoss）无直接关联。

## Used Mertric
In this project, use two evaluation metrics in experiments: **AUC (Area Under ROC)** and **Logloss (cross entropy)**, just as the assay says.
